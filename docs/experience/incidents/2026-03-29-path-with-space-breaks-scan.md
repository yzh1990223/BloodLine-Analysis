# 路径带空格导致扫描失败

## 1. 类型

incident

## 2. 背景

首页扫描控制面板支持手动输入 `.repo` 路径并触发扫描。在本地使用真实样例文件时，路径中包含空格。

## 3. 现象

点击“重新扫描解析”后返回 500，后端报 `FileNotFoundError`。

## 4. 根因

前端或用户输入中可能包含 shell 风格的 `\ ` 空格转义，后端原先直接把字符串原样当路径打开，没有在应用层做归一化处理。

## 5. 影响

- 扫描控制面板在真实路径场景下不可用
- 用户容易误以为是后端服务挂掉

## 6. 处理

后端在扫描输入处理阶段增加了对 `\ ` 到真实空格路径的兼容归一化，并补充了对应验证。

## 7. 可复用结论

- 用户输入路径时，不要假设一定是“纯应用路径格式”
- 对文件路径类输入，服务端应尽量做输入归一化，而不是把 shell 约定泄漏给用户

## 8. 回灌动作

- 文档：已在 README 与使用说明中写明路径可直接传真实路径，兼容 `\ ` 形式
- 测试：已回灌为相关扫描验证
- hooks / CI：不适合直接进入 hook
- AGENTS：不需要进入项目级规则
- backlog：无需进入 backlog
