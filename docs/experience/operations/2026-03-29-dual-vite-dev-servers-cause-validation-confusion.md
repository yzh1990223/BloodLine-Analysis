# 双前端 dev server 导致验证混乱

## 1. 类型

operation

## 2. 背景

在前端持续调试过程中，本地曾同时存在两个 Vite dev server，分别监听不同端口。

## 3. 现象

- 页面看起来“和最新代码不一致”
- 某些交互在一个端口复现，在另一个端口不复现
- 容易误判为前后端逻辑错误

## 4. 根因

验证对象并不唯一。开发者以为在看当前代码，但实际浏览器打开的是旧进程对应页面。

## 5. 影响

- 排障效率下降
- 易产生错误结论
- 会把运行环境问题误判为业务逻辑问题

## 6. 处理

关闭旧前端进程，统一只保留一个明确端口的 dev server，并在排障时先确认当前访问的是哪一个实例。

## 7. 可复用结论

- 真实 UI 验证前，先确认“当前看的页面到底是哪一个进程”
- 本地多实例并存时，环境一致性本身就是一类排查对象

## 8. 回灌动作

- 文档：已回灌到运行和验证说明
- 测试：不适合单元测试化
- hooks / CI：不适合进入 hook
- AGENTS：不需要项目级强规则，但属于运行经验
- backlog：如后续需要，可加入统一启动脚本或开发环境状态检查
