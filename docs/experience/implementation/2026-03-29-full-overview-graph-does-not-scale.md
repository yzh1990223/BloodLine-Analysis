# 首页全量总览图不适合大图场景

## 1. 类型

implementation

## 2. 背景

首页一度尝试默认展示全量对象节点与全部连线，作为系统总览入口。

## 3. 现象

- 节点和边一多，页面拖不动
- 图会乱成一团
- 用户无法快速读图

## 4. 根因

首页承担的是“入口与定位”角色，不是专业图分析工作台。把所有对象级关系默认一次性摊开，会同时遇到：

- 渲染性能问题
- 布局密度问题
- 可读性问题

## 5. 影响

- 首页交互体验明显下降
- 用户难以从总览进入有效分析

## 6. 处理

首页改成：

- 扫描控制面板
- 对象概览
- 搜索
- 选中对象后的局部预览

而不是默认渲染全量大图。

## 7. 可复用结论

- “默认全量图”不是图产品的成熟标志
- 大图问题优先应通过信息分层解决，而不是只靠优化样式和布局硬撑

## 8. 回灌动作

- 文档：已回灌到 README、阶段总结和实现说明
- 测试：页面行为已通过前端测试覆盖
- hooks / CI：不适合进入 hook
- AGENTS：不需要进入项目级规则
- backlog：后续如需高级图工作台，应作为独立方向设计
