# 2026-03 经验 Digest

## 本阶段经验主题

2026-03 阶段，项目里最明显的经验主要集中在 4 类：

1. 输入与运行环境
2. 图展示与可读性
3. 本地验证与开发环境一致性
4. 治理分层边界
5. GitHub Project 任务闭环落地

## 已沉淀的代表经验

### 1. 输入路径与运行环境

- 路径中的空格会让扫描输入在 shell 场景下出现转义差异
- 多个本地 dev server 并存会让 UI 验证对象混乱

结论：

- 运行问题不应只修实现，还应沉淀为“验证路径和启动约束”

### 2. 图展示与可读性

- 首页全量总览图在对象和边增多后会迅速失去可读性

结论：

- 大图场景下，默认不应追求“全量摊开”，而应优先局部预览与逐步聚焦

### 3. 治理体系边界

- AI hooks 和 `superpowers` 在职责上天然接近，但不应重复

结论：

- `AGENTS.md` 管仓库事实
- `superpowers` 管方法
- AI hooks 管事件提醒
- hooks / CI 管门禁

### 4. GitHub Project 任务闭环

- GitHub Project 的父子层级关系必须显式建立，不能依赖命名或 Milestone 让 UI 自己推断
- GitHub 内置 `Status` 不足以表达完整任务流转，需要和自定义 `Workflow` 配合使用
- Project 视图配置不应高估自动化浏览器能力，部分步骤需要人工在已登录浏览器中完成

结论：

- GitHub Project 落地应采用“双层状态模型 + 显式父子关系 + 自动化与人工协同”的策略

## 已发生的回灌

本阶段经验已回灌到：

- 图展示策略
- 治理文档
- `AGENTS.md`
- smoke 校验
