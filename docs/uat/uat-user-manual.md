# BloodLine Analysis UAT 测试操作手册

## 1. 文档目的

本手册面向 UAT 测试人员，用于指导当前 `MVP1.1` 版本的页面操作、链路验证和问题记录。

## 2. 测试前说明

当前版本实际分析输入为：

- Kettle `.repo` 文件
- Java 源码目录

当前尚未真正接入：

- MySQL 元数据实时读取
- 字段级血缘

因此本轮测试重点是：

- 对象级 / 表级血缘是否合理
- 影响分析是否合理
- 闭环分析是否能暴露异常链路
- 页面交互是否顺畅

## 3. 页面入口

请向部署人员确认 UAT 地址，例如：

```text
http://your-uat-domain/
```

主要页面：

- 首页：`/`
- 对象列表：`/objects`
- 闭环分析：`/analysis/cycles`
- 对象详情：`/tables/{table_key}`
- 影响分析：`/tables/{table_key}/impact`

## 4. 首页功能说明

首页是当前版本的主入口，包含以下区域。

### 4.1 顶部导航

导航栏包含：

- `总览`
- `表搜索`
- `闭环分析`
- 面包屑

### 4.2 扫描控制面板

首页顶部有“扫描控制面板”，用于：

- 查看最近一次扫描状态
- 查看最近更新时间与扫描记录 ID
- 手动触发“重新扫描解析”
- 展开“高级配置”

高级配置支持填写：

- `Repo 文件路径`
- `Java 源码目录`
- `MySQL DSN（预留）`

说明：

- `Repo 文件路径` 和 `Java 源码目录` 至少填一个
- `MySQL DSN` 目前不会真正参与解析

### 4.3 对象概览

首页中部是对象概览卡片，显示：

- `总对象数`
- `源表`
- `源文件`
- `数据表`
- `闭环分析`

点击任一卡片都可以进入相应列表或分析页面。

### 4.4 搜索与局部预览

首页下半部分支持：

- 输入关键词搜索对象
- 点击对象名进入详情页
- 点击“预览”查看当前对象的一跳局部血缘图

局部预览图支持：

- 单击节点：高亮当前节点及直接上下游
- 双击节点：进入该对象详情页

## 5. 对象列表页功能说明

对象列表页支持：

- 按对象类型浏览
- 在当前类型内再次搜索
- 查看对象名称、对象类型、对象 key
- 点击对象进入详情页

当前对象类型包括：

- `数据表`
- `源表`
- `源文件`

## 6. 对象详情页功能说明

对象详情页会展示：

- 当前对象名称与类型
- 直接上游对象
- 直接下游对象
- 完整链路图
- 关联对象
  - 作业
  - Java 模块
  - 转换

页面内按钮：

- `返回总览`
- `查看影响分析`

### 6.1 完整链路图交互

详情页中的完整链路图支持：

- 单击节点
  - 黑色边框高亮当前命中节点
  - 高亮其一级上下游节点和相关边
- 双击节点
  - 跳转到该对象详情页

图中展示的是围绕当前对象裁剪后的方向性链路：

- 上游节点仅保留继续向上追溯的链路
- 下游节点仅保留继续向下扩散的链路

### 6.2 关联对象高亮

详情页底部“关联对象”区域支持点击筛选。

例如点击某一个作业时：

- 链路图会高亮该作业涉及的对象节点
- 节点之间的相关边也会高亮
- 其余内容会降噪
- 再次点击同一对象可取消高亮

## 7. 影响分析页功能说明

影响分析页会展示：

- 当前对象名称与类型
- 最多 3 跳下游影响对象
- 每个对象对应第几跳影响

页面内按钮：

- `返回总览`
- `返回表详情`

## 8. 闭环分析页功能说明

闭环分析页用于查看多表闭环组。

页面会展示：

- `闭环组数量`
- `闭环表总数`
- `组内闭环边总数`
- 闭环组列表

当前闭环分析规则：

- 只统计至少 2 张表参与的闭环组
- 单表自环不在该页面分组展示

每个闭环组中可看到：

- 组内表数量
- 组内边数量
- 参与闭环的对象列表

点击对象名可进入详情页。

## 9. 建议测试路径

### 路径一：扫描流程

检查点：

- 首页是否能看到扫描控制面板
- 最近扫描状态是否显示正常
- 重新扫描解析是否可触发
- 扫描完成后首页概览是否刷新

### 路径二：首页入口

检查点：

- 对象概览卡片是否有数据
- 点击卡片是否能进入对应页面
- 搜索是否可用
- 局部预览是否可用

### 路径三：对象详情页

检查点：

- 上下游列表是否合理
- 完整链路图是否加载成功
- 单击 / 双击节点交互是否正常
- 关联对象点击高亮是否生效

### 路径四：影响分析页

检查点：

- 下游对象是否合理
- 跳数是否合理
- 返回按钮和面包屑是否正常

### 路径五：闭环分析页

检查点：

- 首页闭环分析数量与闭环分析页是否一致
- 闭环组列表是否正常展示
- 点击闭环对象是否能进入详情页

## 11. 缺陷记录建议

提交缺陷时建议包含：

- 问题标题
- 页面地址
- 测试对象
- 操作步骤
- 实际结果
- 预期结果
- 截图或录屏

示例：

- 问题标题：点击关联对象后链路图高亮范围不正确
- 页面地址：对象详情页
- 测试对象：`bond_master_data`
- 操作步骤：
  1. 打开该对象详情页
  2. 点击关联作业 `map_windbond`
- 实际结果：高亮范围与其它作业一致
- 预期结果：仅高亮该作业实际涉及的节点和边

## 12. 当前版本已知限制

- 仅支持对象级 / 表级血缘
- `mysql_dsn` 只是预留字段
- 扫描为同步执行，大数据量下响应会较慢
- 复杂动态 SQL、脚本类步骤仍存在覆盖盲区
- 闭环分析当前仅展示闭环组，不枚举每一条具体环路

## 13. UAT 验收建议

建议本轮 UAT 至少确认以下结论：

- 页面主入口、导航和返回路径可用
- 扫描流程可用
- 典型业务对象能查到上下游和影响范围
- 详情页完整链路图可用
- 闭环分析能发现多表闭环组
