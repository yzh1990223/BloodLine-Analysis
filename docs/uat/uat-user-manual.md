# BloodLine Analysis UAT 测试操作手册

## 1. 文档目的

本手册面向 UAT 测试人员，帮助测试人员在当前 BloodLine Analysis MVP 版本中完成：

- 扫描任务触发
- 血缘链路查询
- 页面功能验证
- 问题记录

## 2. 测试前说明

当前版本主要分析以下两类输入：

- Kettle `.repo` 文件
- Java 源码目录

当前版本尚未接入：

- MySQL 元数据实时读取
- 字段级血缘

因此测试重点应放在：

- 表级血缘链路是否正确
- 影响分析是否合理
- 页面交互是否顺畅

## 3. 访问地址

请向部署人员确认 UAT 地址，例如：

```text
http://your-uat-domain/
```

## 4. 扫描任务操作

当前版本的扫描一般由部署人员或测试负责人通过接口触发。

标准请求示例：

```bash
curl -X POST http://your-uat-domain/api/scan \
  -H 'Content-Type: application/json' \
  -d '{
    "repo_path": "/data/bloodline-uat/repo/merged.repo",
    "java_source_root": "/data/bloodline-uat/java-src"
  }'
```

如果你是页面测试人员，通常不需要自己执行这一步，只需要确认：

- 数据已经导入完成
- 首页能看到总览图

## 5. 首页功能说明

打开首页后，可以看到以下区域：

### 5.1 顶部导航

顶部包括：

- `BloodLine`
- `总览`
- `表搜索`
- 面包屑

用于确认当前位置并快速跳转。

### 5.2 全量表级总览图

首页中间区域展示全量表级血缘图。

支持以下操作：

- 单击节点：高亮当前节点及其直接上下游
- 再次单击同一节点：取消高亮
- 点击空白区域：取消高亮
- 双击节点：进入表详情页

### 5.3 搜索区

在首页下方搜索框输入表名关键词，例如：

- `orders`
- `summary`
- `dashboard`

点击 `搜索` 后查看匹配结果。

## 6. 表详情页功能说明

进入表详情页后，可以查看：

- 当前表名
- 直接上游表
- 直接下游表
- 关联作业
- 关联 Java 模块
- 关联转换

页面中可使用：

- `返回总览`
- `查看影响分析`

## 7. 影响分析页功能说明

进入影响分析页后，可以查看：

- 当前表影响到哪些下游表
- 每张表属于第几跳影响

页面中可使用：

- `返回总览`
- `返回表详情`

## 8. 建议测试路径

建议按以下顺序测试：

### 路径一：首页总览图

检查点：

- 页面能否打开
- 总览图是否加载成功
- 节点和边是否能看清
- 单击高亮是否有效
- 双击是否进入详情页

### 路径二：表搜索

检查点：

- 搜索框是否可用
- 搜索结果是否正确
- 点击搜索结果是否进入详情页

### 路径三：表详情页

检查点：

- 上游、下游是否合理
- 关联对象是否合理
- 返回按钮是否可用
- 影响分析入口是否可用

### 路径四：影响分析页

检查点：

- 受影响表是否合理
- 跳数是否合理
- 返回按钮是否可用
- 面包屑是否正确

## 10. 缺陷记录建议

如果发现问题，建议按以下格式提交：

- 问题标题
- 页面地址
- 测试对象表名
- 操作步骤
- 实际结果
- 预期结果
- 截图

示例：

- 问题标题：`ods.orders` 双击节点无法进入详情页
- 页面地址：首页总览图
- 操作步骤：
  1. 打开首页
  2. 双击 `ods.orders`
- 实际结果：页面未跳转
- 预期结果：进入 `ods.orders` 表详情页

## 11. 当前版本已知限制

- 当前只支持表级血缘，不支持字段级血缘
- 当前没有接入 MySQL 元数据直连
- 动态 SQL 覆盖能力有限
- 大规模扫描时接口响应时间可能偏长

## 12. UAT 验收建议

建议本轮 UAT 至少覆盖以下结论：

- 首页查询与总览图功能可用
- 表详情和影响分析查询链路可用
- 样例核心链路正确
- 典型真实场景链路基本正确
- 页面导航和返回路径顺畅
