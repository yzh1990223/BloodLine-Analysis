# 最近扫描输入持久化与自动回填实施计划

## 1. 目标

让首页扫描控制面板能够在点击“重新扫描解析”后持久化当前输入，并在后续重新打开页面时自动回填最近一次扫描使用的输入。

## 2. 实施步骤

### 步骤 1：补后端回归测试

- 扩展 `backend/tests/test_scan_api.py`
- 先写失败测试，覆盖：
  - `POST /api/scan` 持久化 `inputs`
  - `GET /api/scan-runs/latest` 返回 `inputs`

### 步骤 2：补前端回归测试

- 扩展 `frontend/src/__tests__/ScanControlPanel.test.tsx`
- 先写失败测试，覆盖：
  - 最近扫描返回 `inputs` 时自动回填

### 步骤 3：实现后端持久化

- 扩展 `ScanRun` 模型，增加 `inputs`
- 增加 Alembic 迁移
- 扫描创建逻辑保存非空输入
- 最新扫描接口返回 `inputs`

### 步骤 4：实现前端自动回填

- 扩展前端类型定义
- 在 `ScanControlPanel` 首次加载最近扫描时自动设置表单值
- 保持现有“只提交非空字段”的行为

### 步骤 5：验证与收口

- 后端测试
- 前端测试
- 前端构建
- 必要时本地刷新页面验证

## 3. 验收标准

- 重新扫描后，最近扫描记录中能看到提交的输入
- 刷新首页后，扫描面板能自动回填最近一次扫描输入
- 提交空字段时，不会把空值持久化进去
- 现有扫描流程行为不回归
