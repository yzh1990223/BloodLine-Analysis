# 经验沉淀提醒器设计

## 1. 背景

经验沉淀闭环第一版已经建立了：

- 目录结构
- 统一模板
- 索引
- 样例经验
- 治理入口

但当前仍然完全依赖人工判断“这次是否值得沉淀经验”。这意味着体系虽然存在，但容易因为忙碌或注意力转移而被跳过。

## 2. 目标

增加一个第二版轻量提醒器，在 `post-commit` 时根据本次提交内容判断：

- 是否值得沉淀经验
- 更可能属于哪一类经验
- 应查看哪份入口文档

第一版提醒器只做：

- 提醒
- 分类建议
- 入口引导

不做：

- 自动创建经验文件
- 阻断提交
- 复杂评分或状态流转

## 3. 触发时机

触发时机固定为：

- `post-commit`

这样不会打断开发过程，也更适合在提交完成后做一次“是否值得沉淀”的轻量反思。

## 4. 触发规则

第一版命中以下场景时提醒：

1. commit message 以 `fix:` 开头
2. 提交包含：
   - `scripts/hooks/`
   - `.github/workflows/`
   - `AGENTS.md`
3. 提交包含：
   - `backend/src/bloodline_api/services/lineage_query.py`
4. 提交包含：
   - `frontend/src/pages/`

这些规则分别对应：

- `incident`
- `governance`
- `implementation`
- `implementation`

## 5. 提醒内容

命中后输出：

- 为什么建议沉淀
- 建议分类
- 建议入口

建议入口固定为：

- `docs/experience/README.md`
- `docs/governance/experience-closure-foundation.md`

## 6. 设计原则

第一版坚持：

- 只提醒
- 不阻断
- 不自动生成文档
- 不尝试替代人的判断

这样可以保证它是经验闭环的“触发器”，而不是新的负担。

## 7. 成功标准

实现完成后，应达到：

- 普通代码提交后仍能正常完成 `post-commit`
- 命中上述场景时，会额外输出经验沉淀提醒
- 提醒里有明确的经验类别建议
- 提醒里有可直接去看的入口文档
